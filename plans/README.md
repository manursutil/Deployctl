## Repo conventions executors must preserve

- TypeScript with Node.js ESM.
- Tests use Node's built-in test runner via `node --import tsx --test`.
- CLI command controllers stay thin over directly importable core modules.
- Operational YAML config is parsed through strict schemas and represented as typed objects.
- Git access stays behind `RefResolver` / `src/adapters/git.ts`.
- Deploy history/current-state access stays behind `DeployHistoryRepository`.
- Deploy orchestration must set the `CurrentState.inProgress` guardrail before work starts and clear it on completion or failure.
- Secret values must never pass through `deployctl`; only secret references belong in config and orchestration inputs.
- The future dashboard must call the same orchestration modules as the CLI, not shell out to the CLI binary.

## Findings considered and rejected

- Command injection via Git ref/repository args: rejected. The current adapter uses `execFile("git", args, ...)`, not shell command string interpolation. Continue preserving `execFile` or an equivalently non-shell API.
- Hardcoded secrets in checked-in tenant config: rejected for the quick audit. `tenants.yml` contains secret references such as Secrets Manager names, not secret values, and `src/core/tenants.ts` rejects likely secret values.
- High/critical dependency vulnerabilities: rejected. `npm audit --audit-level=high` reported 0 vulnerabilities during the audit.

## Original finding summary

1. `src/core/frontend.ts` records a frontend deploy failure when smoke check returns `false`, but not when `smokeCheck.check()` throws.
2. `src/core/guardrail.ts` implements guardrail acquisition as check-then-write, which is not safe for a future persistent repository under concurrency.
3. `src/adapters/git.ts` accepts a full commit SHA without proving it exists in the configured application repository.
4. `src/core/frontend.ts` ignores configured `build.frontend.buildConfigIdentityInputs` when computing frontend artifact identity.
5. `src/adapters/git.ts` has no direct unit tests; existing ref tests use a fake `RefResolver` and do not cover parsing/selection/error wrapping.
